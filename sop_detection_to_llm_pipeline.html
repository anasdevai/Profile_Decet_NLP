Gaps — what's missing or underspecified
Critical gaps that will reduce LLM output quality
No vocabulary density threshold sent to LLM — it knows terms exist but not their frequency baseline. A term used 40× vs 2× should have different enforcement weight.
Precision term registry built but violation list NOT passed to LLM — LLM must re-detect violations it's supposed to avoid. The pre-computed violation list (Stage 8D) should be injected as a "never use these alternatives" blocklist.
Step completeness scores per step NOT passed to LLM — LLM doesn't know which exact steps scored 1/4 or 2/4. It gets a summary ratio, not the specific deficient steps.
Tone signal RAW values (SHALL_ratio 0.42, passive_ratio 0.28) detected in Stage 3 are NOT injected into the system prompt — LLM can't calibrate to match source document's exact register.
Section semantic labels (purpose/scope/procedure etc.) not sent as a map — LLM rewrites without knowing the semantic role of each chunk, risking role confusion.
CREATE task: no minimum vocabulary seeding — when creating from scratch, LLM has no domain term list to draw from, so output may use imprecise synonyms instead of precision terms.
Bilingual term pairs not explicitly mapped — LLM knows the document is bilingual but has no glossary of EN↔DE (or other) precision equivalents to enforce during translation or rewrite.
